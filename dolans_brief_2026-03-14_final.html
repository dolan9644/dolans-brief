AutoGen 安全事件：协作框架的光谱污染

微软 AutoGen 团队在 3 月初紧急下架了 FunctionTool，起因是一个标准的 RCE（远程代码执行）漏洞。当 LLM 生成的工具调用参数未经充分沙箱化直接送入 eval() 或 exec() 时，攻击者可以通过构造恶意 prompt 诱导模型输出带有系统命令的函数参数。

# 脆弱的函数执行代码 def execute_tool(tool_name, args): if tool_name == "read_file": # 直接拼接执行——经典的注入漏洞 result = eval(f"open('{args['path']}').read()") return result

AutoGen 的核心架构依赖多智能体协作，每个 agent 可以调用其他 agent 的工具。当其中一个 agent 被 prompt 注入控制后，攻击链可以沿着 agent 通信图谱横向移动。

Gemini 3 家族：Deep Think 的暴力美学

DeepMind 发布 Gemini 3 家族：Flash-Lite 面向超低延迟场景（首 token 延迟低于 200ms），Pro 定位于企业级综合任务，Nano Banana 2 是 1.8B 的端侧模型。

杀手锏是 Deep Think 模式。显式的"慢思考"推理引擎，核心机制是动态扩展思维链并引入蒙特卡洛树搜索（MCTS）进行多路径探索。在 IMO 基准测试中达到金牌水平。

# 简化的推理预算分配逻辑 def deep_think(problem, budget_tokens=32000): # 第一阶段：问题拆解 decomposition = decompose(problem, max_tokens=2048) # 第二阶段：多路径蒙特卡洛搜索 solutions = [] for _ in range(candidate_paths): path = mcts_search( problem=decomposition, simulations=500, depth_limit=12 ) solutions.append(path) # 第三阶段：结果融合与验证 verified = verify_and_merge(solutions) return verified

关键参数：`simulations=500`、`depth_limit=12`、`budget_tokens=32000`。在 A100 GPU 上单次推理耗时约 45 秒——相比普通模式的 0.8 秒，慢了 56 倍，但正确率从 67% 跃升至 94%。

LeRobot v0.5.0：物理智能的安卓时刻

LeRobot v0.5.0 是机器人领域的"安卓时刻"——硬件抽象层标准化，软件堆栈模块化、模型权重开源化。

from lerobot import RobotDataset, RobotPolicy, LeRobotConfig config = LeRobotConfig( dataset_path="hf://datasets/lerobot/push_t", policy_name="act", image_resolution=(96, 96), action_space="end_effector", batch_size=64, ) dataset = RobotDataset.load(config.dataset_path) policy = RobotPolicy(observation_type="pixels", action_type="continuous") policy.train(dataset, config)
