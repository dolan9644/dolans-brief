# Codebase Brief — 持久记忆

> 每次 cron isolated session 启动时，首先读取此文件。
> 简报风格和偏好按此文件执行。
> 用户对简报的反馈通过回复简报消息传达给 main session（bibi-agent）。

---

## 简报风格

- **不要编号**（不列 PR 编号）
- **自然语言**，不是给程序员看的 raw PR 列表
- **连接用户体验**：描述用户感受到什么变化，而不是技术改动
- **重要变化多说两句解释背景**
- **分主题输出**，不要罗列所有 PR

---

## 用户关心的方向（按优先级）

1. **界面与交互** — 多会话管理、用量统计、网页端 UI
2. **AI 对话体验** — 超时机制、模型回退、附件加载、多 Agent 设置
3. **系统稳定性** — Discord/飞书/Telegram 消息处理、投票功能、ACP 卡死恢复
4. **Channel 更新** — Telegram / 飞书 / Discord / iMessage / Slack 的支持情况
5. **性能优化** — 启动速度、内存占用
6. **新功能** — 有意思的、可能对使用有帮助的新能力

---

## 不需要提的方向

- 纯工程 CI / 测试框架
- 纯内部架构重构（doctor 重构、SDK 统一等用户无感知的）
- 代码格式、依赖更新

---

## 用户已知 / 遇到过的问题（供结合背景说明）

- 用户不使用 Discord
- 用户对 AI 工具运营和自动化有兴趣
