| 层级 | 工具 | 用途 |

- 429 retry 等 3s 但 API 要求 18s，永远降级（已修 retryAfter）
- 并发 6 线程快速撞墙（已改为 3 线程）

### 改进方向（待实施）
1. HN 改用 Firebase API（`https://hacker-news.firebaseio.com/v0/topstories.json`）
2. 主流网站 BeautifulSoup 直抓（36Kr、Pragmatic Engineer、机器之心）
3. Jina 只做兜底，减少 90% 调用量
4. 全局请求间隔 0.5-1s
